# 🤖 Godot AI Assistant 操作手册 (v1.0)

## 📖 简介
本插件是专为 Godot 4 引擎开发设计的集成式 AI 助手。它通过对接大语言模型（如 DeepSeek），利用 **项目全局感知** 与 **上下文提炼技术**，协助开发者进行代码编写、架构设计及逻辑调试。

---

## 🏗️ 界面布局
插件界面采用垂直流式布局，针对开发者习惯优化，分为三个核心交互区域：

### 1. 顶部：会话管理栏 (Global Session Bar)
* **会话选择器 (Session Selector)**：下拉切换所有历史对话。标题会根据首句提问自动生成，方便识别。
* **新建按钮 (+)**：立即开启一轮全新的对话，旧对话将自动保存至本地存档。
* **删除按钮 (❌)**：永久移除当前选中的会话存档。

### 2. 中间：交互显示区 (Chat Display)
* **多模态显示**：支持 BBCode 渲染，生成的代码块将自动进行语法高亮。
* **推理链展示 (Reasoning)**：支持深度推理模型（如 DeepSeek-R1）。系统会以灰色斜体实时展示 AI 的“思维链”过程，增强逻辑透明度。

### 3. 底部：核心控制栏 (Control Dock)
* **模型切换**：在普通对话 (Chat) 与深度推理 (Reasoner) 模式间一键切换。
* **清空 (🗑️)**：重置当前会话，清除显示内容与内存历史。
* **压缩 (✂️)**：**核心黑科技**。调用 AI 对当前冗长对话进行语义浓缩，生成高浓度“记忆备忘录”，在保留开发背景的同时大幅释放 Token。
* **设置 (⚙️)**：管理 API URL、API Key 及自定义模型版本。
* **插入代码**：一键将 AI 生成的代码片段精准注入到当前编辑器光标所在位置。

## ⚙️ 快速上手

### 1. API 初始化
1. 点击插件底部的 **⚙️** 图标。
2. **API URL**：建议填写官方完整端点：`https://api.deepseek.com/chat/completions`。
3. **API Key**：填入你的有效密钥。
4. **模型名称**：默认为 `deepseek-chat`。（作者使用的是deepseek进行测试）
5. 点击确定后，配置将持久化保存，下次启动无需重复填写。

### 2. 智能上下文感知
发送请求时，插件会自动采集以下信息随 Prompt 发送，AI 将拥有“上帝视角”：
* **项目架构**：自动扫描项目所有的 Autoload 单例及其路径。
* **当前脚本**：实时读取编辑器中正在编辑的代码上下文，实现“针对性”优化。

## 💾 数据存储说明
插件所有本地数据均遵循 Godot 标准存储于 `user://` 目录下：
* **配置文件**：`ai_assistant_config.cfg` (存储 API 敏感参数)
* **对话存档**：`ai_sessions.json` (采用标准 JSON 格式存储多会话数据)

## 🛠️ 故障排除 (Troubleshooting)

| 状态码 | 释义 | 解决建议 |
| :--- | :--- | :--- |
| **404** | 接口未找到 | 检查 URL 是否完整，结尾是否缺少了 `/chat/completions`。 |
| **400** | 请求参数错误 | 检查设置中的“模型名称”是否拼写正确（如 `deepseek-chat`）。 |
| **401** | 身份验证失败 | 检查 API Key 是否填写正确、是否有余额或已过期。 |
| **Timeout** | 请求超时 | 针对 Reasoner 模式，请在 Inspector 检查 `HTTPRequest` 节点的 Timeout 是否设为 120s 以上。 |

---

## 💡 使用建议
* **长对话优化**：当对话轮次超过 10 轮时，建议使用 **✂️ 压缩** 功能，这能显著提升 AI 的响应质量并节省 API 配额。
* **复杂逻辑**：涉及游戏架构设计或数学算法时，请切换至 **Reasoner** 模式。
* **跨平台兼容**：本插件底层支持所有兼容 OpenAI 格式的 API（如 Kimi、通义千问、本地 Ollama 等）。
