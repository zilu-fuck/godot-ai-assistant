# AI Assistant v2.0.1 更新报告

## 版本定位

v2.0.1 是一次面向 `addons/ai_assistant` 的结构化重构版本，重点不在界面换皮，而在于把插件从“Dock 中堆逻辑”推进到“运行时、规则、记忆、Provider、动作执行分层”。

## 本次完成的核心升级

### 1. 运行时主链路拆分

- `ai_dock.gd` 主要负责 UI、会话状态和交互编排
- `core/ai_runtime.gd` 负责发送准备、上下文构建、规则加载、Provider 请求和响应动作规划
- 发送失败、请求预览、上下文占用评估都由 runtime 统一输出

### 2. Prompt / Context / Rules 模块化

- `prompt/ai_prompt_builder.gd` 负责最终消息组装
- `prompt/ai_context_builder.gd` 负责脚本、选区、场景、项目和 Git 上下文构建
- `prompt/ai_message_normalizer.gd` 负责消息归一化
- `prompt/ai_rules_loader.gd` 支持内置规则、`user://ai_assistant_rules.md`、`res://AI_ASSISTANT.md`、目录级 `AI_ASSISTANT.local.md`
- 规则系统支持 `@include`，并能检测循环引用

### 3. 结构化记忆与自动压缩

- `memory/ai_memory_manager.gd` 为 session 引入 `schema_version`、`memory`、`action_log`
- 长会话达到阈值后会自动压缩，只保留最近消息
- 压缩结果会以“结构化记忆”而不是普通聊天历史的形式参与后续请求

### 4. Provider 抽象

- `net/ai_provider_profiles.gd` 统一维护模型档案和能力声明
- `net/ai_provider_adapter.gd` 将统一请求转换成具体 provider payload
- 当前内置支持：
  - `deepseek-chat`
  - `deepseek-reasoner`
  - `openai-chat`
- 对未知模型也支持基于 URL 的兼容推断

### 5. ActionExecutor 收口 Apply 流程

- `actions/ai_action_executor.gd` 统一处理插入、替换、diff 预览和风险确认
- AI 返回代码块后，不再直接写回编辑器，而是先规划动作
- 支持：
  - 无选区时插入到光标
  - 有选区时优先替换选区
  - 命中函数/文件目标时走候选目标与预览流程

### 6. 项目上下文增强

- `project/ai_project_indexer.gd` 提供项目文件摘要和文件匹配能力
- `project/ai_git_context.gd` 提供当前 Git 状态摘要
- 请求可带上当前脚本、选区、场景、项目地图、Git 摘要和记忆摘要

### 7. 网络与错误处理增强

- `ai_net_client.gd` 现在会正确区分：
  - 连接失败
  - 请求发送失败
  - HTTP 4xx/5xx
  - 空响应
  - 中途断流
  - 超时
- 自定义端口 URL 解析已经修复
- Godot 下的流式读取状态判断也已修正
